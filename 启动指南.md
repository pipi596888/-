# 启动指南（本机开发）

本文以 Windows 为主（仓库内提供 `start.bat`），同时给出通用命令。

## 0. 前置条件

- Go：建议 1.21+
- Node.js：建议 18+
- Docker Desktop：用于运行 MySQL/Redis 容器

本项目默认使用以下容器名（脚本会直接 `docker start`）：

- MySQL：容器名 `mysql`，端口 `3306`，用户/密码 `root/root`
- Redis：容器名 `redis`，端口 `6379`

如果你本机容器名不同，请自行调整脚本或手动启动对应容器。

## 1. 初始化数据库（仅首次）

```bash
type tts-backend\sql\init.sql | docker exec -i mysql mysql -uroot -proot
```

验证：

```bash
docker exec mysql mysql -uroot -proot -e "SHOW DATABASES LIKE 'tts_db';"
docker exec mysql mysql -uroot -proot -D tts_db -e "SHOW TABLES;"
```

## 2. 一键启动（Windows）

双击运行 `start.bat`，它会：

1) 启动 `mysql`、`redis`
2) 启动后端：`user-api`(8081)、`voice-api`(8082)、`tts-api`(8083)、`tts-worker`
3) 启动前端：`tts-front`(3000)

启动后访问：

- 前端：`http://localhost:3000`

## 3. 手动启动（更适合调试）

### 3.1 启动后端（4 个终端）

```bash
cd tts-backend/user-api   && go run . -f etc/user-api.yaml
cd tts-backend/voice-api  && go run . -f etc/voice-api.yaml
cd tts-backend/tts-api    && go run . -f etc/tts-api.yaml
cd tts-backend/tts-worker && go run . -f etc/tts-worker.yaml
```

### 3.2 启动前端

```bash
cd tts-front
npm install
npm run dev
```

## 4. 可选：DBHub（Database MCP）

根目录 `docker-compose.yaml` 提供 `dbhub` 服务（默认 `8080`）。启动：

```bash
docker compose up -d dbhub
```

该服务会连接到 `host.docker.internal:3306/tts_db`（确保你的 MySQL 在宿主机 `3306` 端口可用）。

## 5. 常见问题排查

### 5.1 端口被占用（Windows）

```bash
netstat -ano | findstr :8083
taskkill /F /PID <PID>
```

### 5.2 数据库连接失败

- `docker ps` 确认容器在运行
- 检查后端配置文件：`tts-backend/*/etc/*.yaml` 的 MySQL/Redis 连接信息

### 5.3 前端无法访问后端

检查 `tts-front/vite.config.ts` 的代理配置与后端端口是否一致。

### 5.4 `npm run build` 报 `spawn EPERM`

常见于安全软件拦截 `esbuild` 子进程或目录权限不足：

- 将项目目录加入安全软件白名单
- 以管理员权限运行终端后重试

## 6. 停止服务

- 关闭 `start.bat` 打开的终端窗口（或 `Ctrl+C`）
- 停止容器：

```bash
docker stop mysql redis
docker compose down
```
